

# 善物单词 - 产品需求文档（PRD）V3.0

> **文档版本**：V3.0 | **状态**：定稿 | **目标平台**：Android 8.0+ | **分发渠道**：国内主流应用商店

---

## 第一部分：产品概述

### 一、产品背景

善物单词是一款专注高效单词记忆的移动应用。产品围绕“选词书→设计划→开始学习”的核心路径设计，以“主动回忆”为记忆机制，通过用户自评（不认识/模糊/简单）驱动个性化学习节奏。

### 二、产品定位

**Slogan**：善物单词 · 专注每一词

**核心价值**：
1. 词书灵活切换，进度不丢失
2. 每日计划自主调整
3. 进度可视化，成果清晰可见
4. 新词学完自动切复习，学习无断档

### 三、目标用户

| 用户类型 | 特征 |
|----------|------|
| 备考群体 | 四六级、考研、雅思托福备考者 |
| 日常学习者 | 持续提升词汇量的英语爱好者 |

### 四、技术环境

| 项目 | 规格 |
|------|------|
| 最低系统 | Android 8.0（API 24） |
| 目标系统 | Android 13+（API 33） |
| 开发语言 | Kotlin |
| 架构 | MVVM |
| 本地数据库 | Room |
| 网络 | 仅用于广告及媒体资源加载 |
| 存储权限 | 无需申请（词书内置，数据存于私有目录） |

### 五、用户状态

| 类型 | 说明 |
|------|------|
| 游客（默认） | 数据本地存储，无需登录，可使用全部基础功能 |
| 会员 | 购买后解锁高级功能，状态本地缓存+支付渠道验证 |

```
所需天数 = ⌈剩余词数 / 每日新词上限⌉
预计日期 = 今天 + 所需天数
```

### 2.5 学习界面（核心）

#### 2.5.1 会话生命周期

```
会话开始：点击首页“开始学习”
    │
    ▼
会话进行中（学习界面操作 / 后台 < 3分钟）
    │
    ├── 返回首页 → 会话结束，进度保存
    ├── 后台 ≥ 3分钟 → 会话结束，状态重置
    └── 学习队列完成 → 进入完成页
```

**激励解锁有效期**：观看激励视频后，解锁状态仅在当前会话内有效。

#### 2.5.2 页面布局

```
┌──────────────────────────────────┐
│  ← 返回      第 12 / 50 个      │ ← 进度指示
├──────────────────────────────────┤
│                                    │
│          English                   │ ← 单词（大号加粗）
│                                    │
│  美式 ▼  [ˈɪŋɡlɪʃ]   🔊          │ ← 发音区
│                                    │
│  ┌──────────┬──────────┬─────────┐ │
│  │ 不认识   │  模糊    │  简单   │ │ ← 三选一（等宽）
│  └──────────┴──────────┴─────────┘ │
│                                    │
│  ──── 反馈区域（动态展开）────    │
│                                    │
│  （根据选择展示不同内容）          │
│                                    │
└──────────────────────────────────┘
```

#### 2.5.3 学习队列构建规则

```
学习队列构建（进入学习时执行）：
│
├── 1. 取新词（从未学过）
│   └── 最多取 daily_new_limit 个
│
├── 2. 取复习词（next_review_date ≤ 今天）
│   └── 最多取 daily_review_limit 个
│
├── 3. 合并队列
│   ├── 新词耗尽后自动切复习词
│   └── 队列为空 → 直接进入完成页
│
└── 4. 展示顺序：新词先于复习词
```

#### 2.5.4 高级功能定义（付费墙）

| 功能 | 描述 | 展示位置 |
|------|------|----------|
| 分词 | 词根/前缀/后缀拆解 + 含义标注 | “不认识”反馈页 |
| 分音 | 音节拆分 + 逐音节慢速发音 | “不认识”反馈页 |
| 场景配图 | 关联场景图片 | “不认识”反馈页 |
| 影视视频 | 15秒影视片段（带字幕） | “不认识”反馈页底部 |

#### 2.5.5 网络状态联动

| 网络状态 | 高级功能表现 |
|----------|--------------|
| 有网络 | 展示解锁入口（看广告/会员专享） |
| 无网络 | 完全折叠高级功能区域，仅显示基础释义 |

**分支二：点击【模糊】**

```

    │
    ▼

    │
    ▼

    │
    ▼

    │
    ├── 免费用户（有网）：底部显示“🔓 看视频解锁完整解析”

    └── 已解锁用户：展示分词+分音
    │
    ▼

```

**分支三：点击【不认识】（核心变现触点）**

```
点击【不认识】
    │
    ▼
展示基础释义 + 例句
    │
    ├── 免费用户（有网）：
    │   ├── 高级区显示解锁占位
    │   │   ├── [🎬 看广告解锁]（激励视频）
    │   │   └── [👑 会员专享]（跳转购买）
    │   └── 显示输入框（选填）
    │
    ├── 免费用户（无网）：
    │   ├── 高级区完全折叠
    │   └── 显示输入框（选填）
    │
    └── 已解锁用户：
        ├── 完整展示分词+分音+配图+视频
        ├── 视频点击播放（不自动加载）
        └── 显示输入框（选填）
    │
    ▼
展示四个确认按钮 → 选择后下一词
```

#### 2.5.7 解锁机制

| 方式 | 流程 | 有效期 |
|------|------|--------|
| 激励视频 | 点击看广告 → 播放15-30s → 当前词解锁 + 会话内后续全部解锁 | 当前会话 |
| 会员 | 点击购买 → 支付成功 → 全局永久解锁 | 会员有效期内 |

#### 2.5.8 四个确认按钮

| 选项 | 含义 | 下次复习间隔 |
|------|------|--------------|
| 重来 | 完全没记住 | 当日或次日 |
| 困难 | 记忆模糊 | 1-2天 |
| 良好 | 基本记住 | 3-5天 |
| 简单 | 非常熟悉 | 7-14天 |

#### 2.5.9 发音功能

| 功能 | 说明 |
|------|------|
| 发音切换 | 下拉选择 美式 / 英式 |
| 播放 | 点击 🔊 按钮 |
| 引擎 | 系统TTS（离线可用） |

#### 2.5.10 退出确认

点击返回 → 弹窗：“确定退出学习吗？进度将自动保存” → [继续学习] / [确认退出]

```kotlin
fun shouldShowInterstitial(): Boolean {
    if (isMember()) return false
    if (getTodayDate() != getLastInterstitialDate()) {
        if (System.currentTimeMillis() - getLastInterstitialTime() >= 6 * 3600 * 1000) {
            return true
        }
    }
    return false
}
```

## 第四部分：业务规则

### 4.1 每日计划计算

```
今日待学新词数 = min(daily_new_limit, 词书剩余新词数)
今日待学复习词数 = min(daily_review_limit, 到期复习词数)
今日总待学 = 今日待学新词数 + 今日待学复习词数
预计用时(分钟) = 新词数 × 0.5 + 复习词数 × 0.167（即10秒/词）
```

### 4.2 进度计算

```
已学词数 = SELECT COUNT(*) FROM LearningRecord WHERE book_id = current_book
进度(%) = 已学词数 / 总词数 × 100
```

### 4.3 复习到期判定

```
当前时间 ≥ next_review_date → 加入复习池
next_review_date 计算规则：
  - 重来：次日
  - 困难：+2天
  - 良好：+5天
  - 简单：+14天
```

### 4.4 连续天数计算

```sql
-- 伪代码
连续天数 = 从今天往前倒推，连续有学习记录的天数
中断判定：某天无学习记录 → 连续中断，次日重新计数
```

### 4.5 输入比对容错

```kotlin
fun compareInput(userInput: String, correctAnswer: String): Boolean {
    val normalizedInput = userInput.trim().lowercase()
    val normalizedAnswer = correctAnswer.trim().lowercase()
    return normalizedInput == normalizedAnswer
}
```
