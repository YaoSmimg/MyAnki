## 第四部分：业务规则

### 4.1 每日计划计算

```
今日待学新词数 = min(daily_new_limit, 词书剩余新词数)
今日待学复习词数 = min(daily_review_limit, 到期复习词数)
今日总待学 = 今日待学新词数 + 今日待学复习词数
预计用时(分钟) = 新词数 × 0.5 + 复习词数 × 0.167（即10秒/词）
```

### 4.2 进度计算

```
已学词数 = SELECT COUNT(*) FROM LearningRecord WHERE book_id = current_book
进度(%) = 已学词数 / 总词数 × 100
```

### 4.3 复习到期判定

```
当前时间 ≥ next_review_date → 加入复习池
next_review_date 计算规则：
  - 重来：次日
  - 困难：+2天
  - 良好：+5天
  - 简单：+14天
```

### 4.4 连续天数计算

```sql
-- 伪代码
连续天数 = 从今天往前倒推，连续有学习记录的天数
中断判定：某天无学习记录 → 连续中断，次日重新计数
```

### 4.5 输入比对容错

```kotlin
fun compareInput(userInput: String, correctAnswer: String): Boolean {
    val normalizedInput = userInput.trim().lowercase()
    val normalizedAnswer = correctAnswer.trim().lowercase()
    return normalizedInput == normalizedAnswer
}
// 注：V1.0 仅做完全匹配，不涉及同义词
// 无论如何都展示正确答案供对照
```
